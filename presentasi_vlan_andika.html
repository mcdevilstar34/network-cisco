BAB I · Pendahuluan
Latar Belakang
Masalah pada Jaringan Tanpa Segmentasi
  • Komputer tamu dapat mengakses perangkat admin secara langsung
  • Data penting mudah diakses oleh user yang tidak berwenang
  • Koneksi lambat akibat broadcast traffic yang berlebihan di satu jaringan
  • Sulit membedakan jaringan guru, siswa, dan tamu
  • Penyebaran virus lebih cepat karena semua perangkat dalam satu segmen
Ancaman: Tanpa pemisahan jaringan, user tidak berwenang bisa mencoba mengakses file atau perangkat lain — termasuk komputer Tata Usaha dan guru.
Solusi yang Diterapkan
VLAN (Virtual Local Area Network) — memisahkan jaringan menjadi kelompok Admin dan Guest meski menggunakan satu switch fisik yang sama, sehingga keamanan dan manajemen jaringan meningkat signifikan.
BAB I – II
Rumusan Masalah & Tujuan Proyek
Rumusan Masalah
  • Bagaimana memisahkan jaringan Admin dan Guest agar tidak dalam satu akses?
  • Bagaimana mencegah user Guest mengakses jaringan Admin?
  • Bagaimana menerapkan keamanan jaringan sederhana menggunakan perangkat Cisco?
Tujuan Proyek
  • Membuat jaringan terpisah antara Admin dan Guest
  • Mencegah user tamu masuk ke jaringan internal
  • Mengurangi gangguan broadcast traffic
  • Mempermudah pengelolaan berdasarkan jenis pengguna
  • Menerapkan VLAN + ACL sebagai solusi efektif
  • Menjadi referensi untuk sekolah, kantor, atau warnet

BAB IV
Fungsi Perangkat & OSI Layer
Router Cisco 2911
  • Melakukan Inter-VLAN Routing antar jaringan
  • Berfungsi sebagai gateway untuk setiap VLAN
  • Menjalankan aturan ACL (Access Control List)
OSI Layer 3 — Network
Switch Cisco 2960
  • Menghubungkan semua perangkat LAN
  • Membuat dan mengelola VLAN (access port)
  • Mengirim traffic multi-VLAN lewat trunk port
OSI Layer 2 — Data Link
PC Client & Kabel
  • Mewakili user Admin (PC0) dan Guest (PC1) dalam simulasi jaringan
  • Digunakan untuk pengujian koneksi (ping) dan verifikasi keamanan ACL
  • Kabel straight-through menghubungkan PC ke Switch dan Switch ke Router
OSI Layer 1 – 7 (Physical hingga Application)

BAB VI · Implementasi
Konfigurasi Switch
Membuat VLAN
enable
conf t

vlan 10
name ADMIN

vlan 20
name GUEST
Assign Port ke VLAN
interface fa0/1
switchport mode access
switchport access vlan 10

interface fa0/2
switchport mode access
switchport access vlan 20
Konfigurasi Trunk Port
interface fa0/24
switchport mode trunk
Port trunk (Fa0/24) memungkinkan semua VLAN melewati satu link fisik menuju router.
BAB VI · Implementasi
Konfigurasi Router & ACL Keamanan
Subinterface VLAN 10 (Admin)
interface g0/0
no shutdown

interface g0/0.10
encapsulation dot1Q 10
ip address 192.168.10.1 255.255.255.0
Subinterface VLAN 20 (Guest)
interface g0/0.20
encapsulation dot1Q 20
ip address 192.168.20.1 255.255.255.0
ACL — Blokir Akses Guest ke Admin
! Tolak traffic dari VLAN Guest (20) menuju VLAN Admin (10)
access-list 100 deny ip 192.168.20.0 0.0.0.255 192.168.10.0 0.0.0.255
access-list 100 permit ip any any

! Terapkan pada subinterface VLAN 20 arah masuk
interface g0/0.20
ip access-group 100 in
Keamanan: ACL diterapkan di sisi Guest (g0/0.20 in) sehingga traffic dari jaringan Guest diblokir sebelum mencapai jaringan Admin.

BAB VIII
Kesimpulan
Hasil Akhir Proyek
  • Sistem keamanan jaringan berbasis VLAN berhasil diterapkan menggunakan Cisco Packet Tracer
  • Segmentasi user Admin (VLAN 10) dan Guest (VLAN 20) berjalan dengan baik pada satu switch fisik
  • ACL berhasil memblokir akses Guest ke jaringan Admin sepenuhnya
  • Inter-VLAN Routing berfungsi normal — Admin dapat menjangkau Guest, tidak sebaliknya
VLAN merupakan solusi yang sederhana namun efektif untuk keamanan jaringan. Dengan kombinasi VLAN dan ACL, keamanan data serta manajemen pengguna dapat ditingkatkan secara signifikan tanpa membutuhkan perangkat tambahan.
Penerapan yang Direkomendasikan
  • Laboratorium komputer sekolah — pisahkan jaringan guru dan siswa
  • Kantor kecil — pisahkan jaringan karyawan dan tamu WiFi
  • Warnet / usaha rumahan — pisahkan jaringan operasional dan pelanggan
